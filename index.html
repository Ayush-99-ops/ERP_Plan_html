Month 3 — Customization & Reports
WK 09–10
Custom App
Custom Frappe App + Workflows
Jul 27–Aug 9
▾

Key Activities

  • Create and maintain a dedicated custom Frappe app on Git
  • Add agreed custom fields to existing doctypes
  • Configure workflows and approval chains
  • Customize quotation format and document templates
  • Customize project and job stages to match operations

What the Custom App Is

  • A Frappe app — Python/JS, not a mobile application
  • Installed on the same server as ERPNext
  • Contains only DVS-specific fields, workflows, print formats
  • Doesn't touch ERPNext core — safe to update later

Month 5 — Production Deployment, Training & Go-Live
WK 17–18
Production
Production AWS Setup + Final Deployment
Sep 21–Oct 4
▾

Key Activities

  • Set up production AWS environment — separate from staging
  • Configure SSL, HTTPS, domain/subdomain, S3 backups, and CloudWatch monitoring
  • Deploy ERPNext production with all custom apps included
  • Import final agreed data — employees, clients, projects
  • Configure production users and permissions

Things to Keep in Mind

  • Staging is for testing — production is where the team works day to day
  • No development or testing directly on production
  • Final data import is a planned, careful operation — not rushed
  • Review AWS cost optimisation once usage stabilises
